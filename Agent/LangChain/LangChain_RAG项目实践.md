# RAG 项目实践

## 先给定位

RAG 项目实践要同时看离线入库和在线问答。只做一个能回答的 demo 不难，难的是让系统长期稳定、可评测、可追踪、可维护

完整 RAG 链路可以分成两条线：离线把资料变成可检索知识库，在线把问题变成可验证答案

## 离线入库

1. 确定知识源和权限边界
2. 做文档解析、清洗和切分
3. 生成 embedding
4. 写入向量库
5. 保存元数据、来源和版本
6. 建立增量更新和删除机制

## 在线问答

1. 接收用户问题
2. 做 query 改写或扩展
3. 向量检索和关键词检索
4. metadata filter 过滤权限和范围
5. reranker 重排
6. 拼接上下文
7. 调用模型生成答案
8. 返回引用和证据

## 调优顺序

不要一上来就调 prompt。先判断问题出在召回、排序、上下文拼接还是生成阶段

- 没召回：检查切分、embedding、检索参数和 query 改写
- 召回错：检查混合检索、过滤条件和 rerank
- 上下文乱：检查 chunk 合并、去重和排序
- 生成差：再调整 prompt、模型和输出约束

## 评测意识

RAG 评测至少要覆盖命中率、引用准确性、答案忠实度、拒答能力和延迟成本。没有评测集的 RAG 优化，大多是在凭感觉调参
