# Memory 记忆存储

## 先给定位

Memory 是让应用保存会话状态或长期偏好的机制。它解决的是“模型每次调用本身没有持久记忆”的问题

模型像一个只看当前输入的函数，记忆系统负责把历史、偏好、摘要或用户画像重新放回上下文

## 两类记忆

- 短期记忆：当前会话里的消息历史、工具结果和中间状态
- 长期记忆：跨会话保存的用户偏好、事实、项目背景和可复用知识

在现代 LangChain 体系里，复杂状态更多会交给 LangGraph 的状态和持久化机制处理，而不是只依赖早期的简单 memory 类

## 为什么不能无限塞历史

上下文窗口是有限资源。把全部聊天记录原样塞回 prompt，会带来成本上升、延迟变高、无关信息干扰和隐私风险

常见策略包括：

- 保留最近若干轮对话
- 对旧消息做摘要
- 把长期事实写入外部存储
- 需要时通过检索取回相关记忆
- 对敏感信息做权限和生命周期控制

## 数学直觉

无记忆调用可以写成：

$$
y_t = f(x_t)
$$

带记忆调用更接近：

$$
y_t = f(x_t, m_{t-1})
$$

其中 `m` 是历史状态。工程难点在于记什么、何时写入、何时取回、何时遗忘
